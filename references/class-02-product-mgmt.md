## 特色选项

### 产品类型 (conf=0.30, 不可逆) force
- **B2C 消费者产品**
  `"## 用户画像\n目标用户：25-35岁一线城市职场女性\n## 核心场景\n用户在下班通勤途中打开APP…"`
- **B2B 企业软件**
  `"## 角色权限\n| 角色 | 查看 | 编辑 | 审批 |\n| 普通用户 | ✓ | ✓ | - |\n| 管理员 | ✓ | ✓ | ✓ |"`
- **内部工具**
  `效率提升指标、审批流、操作日志`
- **平台型产品**
  `双边市场、供需两端分别定义`

### 分析方法论 (conf=0.70, 可逆) 【自动匹配】
- **SWOT 四象限** → 竞品分析/战略规划
- **Porter 五力** → 行业分析/进入评估
- **功能点对比矩阵** → 竞品功能对比
  `"| 功能 | 本品 | 竞品A | 竞品B |\n| 暗色模式 | ✓ | ✓ | - |"`
- **用户旅程地图** → 体验分析
- **北极星指标拆解** → 数据复盘

### 输出模板 (conf=0.60, 可逆) → skip — PRD专属
- **完整正式模板**（背景→目标→功能清单→交互说明→数据埋点→验收标准→排期）(default)
- 精简模板（目标→核心功能→验收标准）

### 数据来源策略 (conf=0.80, 不可逆) → display
- **基于用户提供的实际数据** (default)
- 需要AI基于行业benchmark估算（标注"估算"）

## 盲区预警

- 竞品信息时效性: 所引用竞品信息可能已有更新
- 内部数据: 假设用户提供的数据完整准确
