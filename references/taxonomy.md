# 任务分类决策树

Agent 使用此文件快速将用户请求归入 12 大类，然后只加载对应类的预设文件。

## 分类决策流程

```
用户请求
    │
    ├─ 包含 合同/条款/合规/法律/法条/纠纷 → 3.法务合规
    ├─ 包含 PRD/需求文档/用户故事/路线图/竞品 → 2.产品管理
    ├─ 包含 研报/公司深度/行业分析/尽调/估值 → 1.商业报告
    ├─ 包含 凭证/分录/汇算/纳税/增值税/审计 → 6.财税会计
    ├─ 包含 通知/请示/纪要/决议/公文/周报 → 4.行政公文
    ├─ 包含 论文/文献/引用/审稿/基金/学术 → 8.学术研究
    ├─ 包含 招聘/绩效/薪酬/培训/HR/员工 → 7.人力资源
    ├─ 包含 文案/营销/公众号/小红书/演讲稿 → 5.市场文案
    ├─ 包含 代码/部署/审查/架构/README/CI → 9.代码技术
    ├─ 包含 提醒/日历/日程/会议安排 → 11.日程提醒
    ├─ 包含 图片生成/视频生成/搜索/天气/RSS → 12.API代理
    ├─ 包含 转换/PDF转/OCR/格式转换 → 10.转换处理
    │
    └─ 无法匹配 → 使用通用6维度基线，标注"未归类"
```

## 边界情况处理

### 跨类任务
用户请求可能跨越多个类（如"帮我审这份投资合同" → 法务+商业报告）。处理规则：
- 取第一个触发关键词对应的类为主类
- 在确认单中标注"该任务同时涉及 [副类名称]"
- 默认值以主类为准

### 未归类任务
决策树所有分支都不匹配时：
- 使用通用6维度基线（风格/长度/格式/深度/受众/语言）
- 标注类别为"未归类"
- 交互中使用更高比例的低置信度标记（鼓励用户修正）
- 任务完成后尝试匹配最接近的大类作为下次参考
