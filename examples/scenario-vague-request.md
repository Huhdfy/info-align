# 示例场景：象限❸极度模糊需求

用户："帮我分析一下这个。"

（未提供任何材料、主题、分析维度。）

---

## Agent 分析

- 任务类型：无法归类
- 触发原因：极度模糊——没有主题、没有数据、没有分析目标
- 主要象限：❸模糊消解（用户自己都不确定要分析什么）

## Agent 不直接拒绝，而是维度展开

---

我看到你想做分析，但我手头还没有具体信息。没关系——我先帮你理清"分析"通常涉及哪些方面，你看看哪些符合你的想法：

## 分析类型

1. **竞品分析** — 对比几个产品的功能/定价/市场策略
   你需要提供：竞品名称或列表
   示例输出：功能对比矩阵 + SWOT + 差异化策略

2. **行业分析** — 研究一个行业的市场规模、竞争格局、趋势
   你需要提供：行业名称
   示例输出：行业全景报告（PESTLE + 五力 + 产业链）

3. **财务分析** — 分析一家公司的财务健康状况
   你需要提供：公司名称或财务报表
   示例输出：营收拆解 + 利润率分析 + 杜邦分析

4. **市场策略分析** — 拆解定价/渠道/品牌/推广
   你需要提供：公司或产品名称
   示例输出：4P分析 + 渠道结构图 + 定价带分析

5. **用户反馈分析** — 从用户评论/问卷中提取洞察
   你需要提供：反馈数据（Excel/CSV/文本）
   示例输出：情感分析 + 高频痛点 + 改进建议

6. **其他** — 你说的"分析"可能不在上面，请描述你想分析什么

---
选择1-6中的一个，或直接告诉我你想分析的具体内容。
如果选了某一项但缺材料，我会告诉你需要补充什么。
---

## 关键设计点

1. 不给空白追问"你要分析什么"——这对用户没帮助
2. 展开分析类型的6种常见形态，帮用户在对号入座中理清自己的需求
3. 每种形态附带"你需要提供什么"和"示例输出"——降低用户的认知成本
4. 如果用户选了不在列表中的，就用"其他"通道继续追问
