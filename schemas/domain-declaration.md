# 领域 Skill 信息对齐声明规范

下游 Skill 如何声明自己的信息需求，供信息对齐 Skill 在运行时合并。

## 声明位置

在领域 Skill 的 SKILL.md 中，可选择性包含以下区块（可选——若无声明，信息对齐 Skill 仅使用通用层基线）。

## 声明格式

```markdown
## 信息对齐声明

### 归属大类
12大类之一，如 "商业报告" / "产品管理" / "法务合规" …

### 必须确认项（不可默认，由本 Skill 要求）
- 字段名: 说明 | 用户需提供
- 另一字段: 说明 | 格式要求

### 可默认项（可由信息对齐 Skill 预设）
- 字段名: 默认值 | confidence | 可逆/不可逆
- 另一字段: 默认值 | confidence | 可逆/不可逆

### 盲区预警（前提条件清单）
- 前提条件: 说明 | 默认值（如有）
- 另一前提: 说明 | ⚠️ 需要用户主动确认

### 覆盖通用层规则（覆盖 12 大类预设中的项）
- 通用维度名: 通用默认值 → 本领域新默认值 | confidence | 原因
```

## 声明示例（邀请函 Skill）

```markdown
## 信息对齐声明

### 归属大类
行政公文 — 子标签：活动策划

### 必须确认项
- 收件人: 姓名+称谓 | 如 "张三先生" 或 "李四总"
- 活动时间: 精确到日期+时段 | "7月15日下午2:00-5:00"
- 活动地点: 完整地址 | "XX酒店3楼会议厅"
- 活动主题: 一句话描述 | "2026年夏季新品发布会"

### 可默认项
- 风格: 商务正式 | 0.85 | 可逆
- 长度: 200字 | 0.70 | 可逆
- 是否含RSVP: 是 | 0.60 | 可逆
- 是否含附件(议程/地图): 是 | 0.50 | 可逆

### 盲区预警
- 着装要求: 默认"商务休闲" | ⚠️ 如有特殊要求请说明
- 是否需要多语言版本: 默认"否"
- 是否需要纸版邮寄: 默认"否"

### 覆盖通用层规则
- 格式: 通用层默认"公文格式" → 本领域"邀请函标准格式" | 0.80 | 邀请函有独立排版惯例
```

## 合并优先级

```
领域声明值 > 12大类预设值 > 通用6维度基线值
```

- 领域声明中明确写了"覆盖通用层规则" → 使用领域值
- 领域声明中未写的维度 → 使用 12 大类预设
- 12大类预设中也未定义的 → 使用通用6维度基线
