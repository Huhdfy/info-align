# info-align

> 不逐条追问。预填所有维度，用户只改不对的。

## 一句话

AI 面对模糊指令时不会追问——它直接猜。猜错了，整个结果推倒重来。info-align 在动手前把缺口摊在桌面上，一次确认，不返工。

## 没有它的时候

```
用户: 帮我写一份宁德时代深度研报

AI 在心里默默补全了所有缺口：
  格式 → 应该是 Word 吧
  维度 → 财务+竞争+估值，标准研报套路
  篇幅 → 15页差不多
  风格 → 专业术语拉满
  对标 → 比亚迪和LG，行业惯例

      ↓ 20分钟后

AI: [一份 15 页的 Word 版深度研报，专业但冗长]

用户: 我要的是飞书文档，给老板看的 5 页简报，
      重点是竞争格局，不是财务建模……
      [推倒重来]
```

AI 没有问——它在每一个分叉口都选了一条路。用户拿到的是一个**碰运气的黑箱结果**：缺口越多的任务，AI 盲猜的组合爆炸越严重，第一版就命中的概率越低。

## 有它的时候

```
用户: 帮我写一份宁德时代深度研报

AI: ✅ 标的: 宁德时代 | 报告类型: 深度研报
    ❓ 覆盖维度 → 财务+竞争+估值 (推荐) | 仅财务分析 | 仅竞争格局
    ❓ 输出格式 → PDF (推荐) | Word | 飞书文档
    ⚠️ 估值方法 → DCF (默认) | 可比公司 | 两者
    ⚠️ 对标公司 → 比亚迪、LG新能源 (默认)
    ⚠️ 报告篇幅 → 15-20页 (默认) | 8-12页 | 5页简报

用户: [一键确认]
AI: 开始执行。
```

**一次交互。** 用户做的是审核，不是填空。每一行都是 AI 原本会沉默代入的假设——现在它被摊开了。

## 怎么做到的

不列空白表单让用户填，也不沉默地猜。用四象限法定位四种缺口，每种用不同策略：

| 用户状态 | AI 会犯的错 | 对策 |
|---------|-----------|------|
| 以为 AI 知道，没说出来 | 盲猜一个值，用户拿到结果才发现不对 | 预填推荐值，用户一眼看出有没有偏差 |
| 知道但懒得全说 | 选"最常见"的，但不一定是用户这次要的 | 预填最常见选项，用户跳过即认可 |
| 自己也不确定要什么 | 替用户选一个方向，跑偏了整段白做 | 展开维度帮用户聚焦，不定方向不执行 |
| 不知道这里有坑 | 默认值在特定场景下出错，用户不知道要改 | 预填默认值同时标注风险 |

每个维度 3 个互斥选项，推荐值预填，风险项标注。用户的工作量从"我逐一告诉你"变成"我纠正不对的"。

## 它是什么

一个 meta-skill。不直接执行任务——在**用户和下游技能之间**插一层对齐。输入是用户的模糊指令，输出是 `ResolvedFields[]` 结构化键值对，下游按 `key` 取值直接执行。

## 何时触发

默认对所有任务触发。典型触发信号：

- 动词本身多义：分析、优化、设计、写一个、做一下
- 名词未消解：指代不明、专有名词未定义
- 维度开放：没说格式、受众、范围、工具

明确豁免：纯事实查询、读文件/展示内容、步骤级指令。
